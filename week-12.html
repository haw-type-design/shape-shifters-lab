Organizing content and using grids

This week we will take a look at slightly advanced ways on how you can organize your content using grids and/or the flex layout.
Sometimes in the organisation of your layout you might need more than one column or more advanced options of layout and where your content can go. That’s where grid and flex layouts come in to help. We can think of grids as we would in layout software. To use grids we have to make use of the parent/child concept. As a quick reminder we can nest content inside each other be enclosing the tags into one another like so:
<div class="grid-container">
<div class="grid-item">
</div>
<div class="grid-item">
</div>
<div class="grid-item">
</div>
</div>
In this example grid-container is our parent and the three grid-item our children. So if we define grid-container to be a grid everything inside this tag will be layout inside this grid (we can still define what goes where and even define exceptions). But first of all let’s set up a grid:
.grid-container {
display: grid;
grid-template-columns: 1fr 1fr 300px;
}
 We now defined a grid which has three colums. The rightmost colum is 300px wide. The two colums to the left equally share the space that is left. Let’s give our columns some color, so that we can see what happens:
.grid-item {
background-color:red;
width:100%;
height: 100%;
display: block;
}

.grid-item:nth-child(even) {
background-color:blue;
}
 Here I am making use of the :nth-child pseudo-class to select the second element in our grid (and all multiples of two), so our second column to color it blue.
We can not only define columns but rows, too. If we do not define our rows specifically, once a row is full, adding more elements will result in new rows with the same column layout as the previous. You can try this by adding grid-item elements in the above example. We can now add grid-template-rows to the mix. With this we can define the height of our rows. One example could be:
.grid-container {
display: grid;
grid-template-columns: 1fr 1fr 300px;
grid-template-rows: 50vh 320px;
}
 Here our first row would take 50% of the height of the viewport (normally your screen) and the second 320px. Depending on how large we set our grid-item elements to be, we will have some overlap if they are bigger than our rows. (Luckily we set them to 100% above so they take the width and height defined by their parent, so the grid rows and columns).
Bringing it all together we can also make our grid slightly more human readible by defining the areas and giving them names. This is done using the grid-template rule.
.grid-container {
display: grid;
grid-template:
'title title title'
'notes content content'
'notes footer copyright';
}
 We can then define what goes where by simply adding a grid-area rule in the child items like so: .grid-item-1 {
grid-area: notes;
}
 imagining our grid-item-1 element would hold our notes.

When to flex?

Phew. A lot to digest. What do I need flex for now, I hear you ask. Well the boundaries are fluent between the two. However, somentimes you just want to put your elements neatly aligned next to one another, flowing from row to row without having to think about grid layouts, spaces, rows, columns and what not. Time to flex. Just define your parent element to be flex and the children will flow into the available space like so:
.flex-container {
display: flex;
}
 Sometimes this is just simpler to align your content or distribute it within available space. You are now nicely, equipped with ideas in how to organize your content.
