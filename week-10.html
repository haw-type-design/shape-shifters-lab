Learning a new (Hypertext Markup) Language (HTML)

Exciting! We will learn a new language. And one that machines can speak as well. This one is called HTML which is short for Hypertext Markup Language. More about it here. But let’s start with a bit of terminology to know what we are talking about:
The first important concept is that of tags. These are used to structure our content and to name its elements. The name of such an element is set in angle brackets like: <p> for paragraph. Tags are always opened and closed around their content, just like brackets. You close them using a slash before the tag name: </p>.
These tags can be compiled to elements. Elements comprise and opening and a closing tag enclosing some text content i.e. <p> Some text </p>.
These elements can be nested inside one another to create hierarchy. So for example could each product on a page have a title and a price. The resulting code could look like that:
<product>
<h1 class="product-name">Product name</h1>
<p id="this-price">Product price</p>
</product>
Note that elements are hierarchical. That means that every parent such as <product> which enloses both <h1> and <p> inherits its styling (we will touch this later) to its children (so <h1> and <p> in our case.)
So far so good. We now have a first idea about how to structure our documents. Let’s give our elements some style. Normally this is done using CSS which is the acronym for Cascading Style Sheet. A style declaration consists of a selector (the targeted element, i.e. p), a property (what you’re styling, i.e. font-size), and value (how you’re styling it, i.e. 1em). A stylesheet for our above example could thus look as follows:
product {
background-color: blue;
display: block;
}

.product-name {
font-size: 2em;
color: red;
}

#this-price {
font-weight: bold;
}
Note that different selectors are at play here. We are targeting a tag with product by referencing its name. .product-name is referencing a class. With classes you can apply sets of stylings to multiple HTML elements that have the same class attribute (i.e. <h1 class="product-name">Product name</h1>). It is also possible to apply more than one class by separating them with a space (i.e. <h1 class="product-name some-other-class">Product name</h1>). But be careful that your styling is not contradictory (i.e. it makes no sense to style the font-size twice).

Embedding (variable) fonts

The trick about this — as this one is about styling — lies in CSS. To specify a font, we will make use of the @font-face rule. We can directly dump it into our CSS like so: @font-face {
font-family: "MyFont";
src: url("https://haw-type-design.github.io/octagon-variable/fonts/OctagonVariableVF.woff2");
}
 Now you can use your font as MyFont inside your code. Like so: font-family: MyFont;
Let’s get to the fun part: playing with the variable axes. This is done using the font-variation-settings rule. For our example this could look like this:
.vfont {
font-family: "MyFont";
font-variation-settings: "VROT" -30, "HROT" 30;
}
 Here we make use of the two variable axes of the font VROT and HROT and assign a value one the axis. Here these are -30 and 30.
These are about the basics, that we will need to proceed. Please have a look at the links below and start to experiment. Also don’t forget to use the browser inspector (⌥⌘I) to look at sites that you find interesting and have a look at their code, to see what CSS they use. Experiment and have fun.
