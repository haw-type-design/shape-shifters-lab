Setting up the content structure

This week we will do two things: set up our content structure and dive deeper into animations. But first things first.
For our texts we will need to think a bit like we could think when we were working in a layout software. So thinking about content in terms of headlines, sub-headlines, pull-out quotes, body text, etc.. Depending on your fonts, we might need to decide as a group, if we want to use, say, one additional font for body texts, if the fonts are less suited for small sizes. So let’s discuss that in group this week. But back to layout.
We can think of this like organizing a page in layout software. So we can start to give headlines a class, subheadlines, pull-out quotes and everything else as well. Then we can start to assemble everything by putting the text between the corresponding tags.
Once we got that set up, let’s start to think about what goes where and how we want to style it.

Animations

A thing that we can quickly take a look into (advanced reading on this down below) are animations. Most of the things that we can assign as CSS values are animatable. We can make use of this to animate our variable fonts, using a technique called keyframes. We could set up a simple animation as follows: <div class="vfont"> Headline </div> as our HTML markup using the vfont class to animate our font.
We could add in some styling using CSS like so:
.vfont {
font-family: "MyFont";
animation: my-animation 2s infinite alternate;
}

@keyframes my-animation {
0%{
font-variation-settings: "VROT" -30, "HROT" 30;
}
100%{
font-variation-settings: "VROT" 30, "HROT" -30;
}
} With the first part we set the animation to be my-animation and play forawrds and backwards imfinitely. One animation cycle will have a duration of 2 seconds. In the next step I set the keyframes for my animation using the @keyframes rule. At 0% my font will be animated to "VROT" -30, "HROT" 30 and to "VROT" 30, "HROT" -30 at the end of the two seconds and vice versa. I can also add in intermediate points at, say, 50% so 2 seconds in our case (careful with easing functions this might alter the timeline, see »Further reading«).

Mouse interactions and pseudo-classes

Let’s say we want to perform the same animation, but only when I hover over the font using my mouse. This can be achieved using the :hover pseudo-class. These come in handy when you want to select specific elements in your HTML markup. So for example the element that you just hovered or the nth element in a list. We can just add the wanted behaviour in our CSS like so:
.vfont {
font-family: "MyFont";
}

.vfont:hover {
animation: my-animation 2s infinite alternate;
}

@keyframes my-animation {
0%{
font-variation-settings: "VROT" -30, "HROT" 30;
}
100%{
font-variation-settings: "VROT" 30, "HROT" -30;
}
}
In that case my animation only is played, when the .vfont class gets hovered.
Another way to achieve this (maybe even better for the case of hover animations) are transitions. A transtition controls the speed and the way two states of one class are animated. For our above example this would look as follows:
.vfont {
font-family: "MyFont";
font-variation-settings: "VROT" 30, "HROT" -30;
transition: 2s;
}

.vfont:hover {
font-variation-settings: "VROT" -30, "HROT" 30;
} Important to note is that each animated property is given a value in bothe the .vfont class and also its :hover pseudo-class. Other than that transitions work more or less equivalent to animations.
