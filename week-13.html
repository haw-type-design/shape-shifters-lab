Let's (Java)Script it!

Here we are once more, contemplating the might-this-be-possibles and what-ifs of our newly learned skills. In a next step, let’s start to extend them and add some interactivity and other possibilities to the mix.
Enter JavaScript.
But let’s start it slow. JavaScript will allow us — consciously omitting some of its capabilities — to chime in, where HTML and CSS reach their limits. What could this mean? This could mean querying user interactions such as scrolling, moving the mouse, tilting the device, the microphone, etc. and doing stuff with it such as playing animations, changing CSS classes, changing the content of your page, you name it.Sounds fun huh? So how do we do the magic?
Similar to CSS, our JavaScript needs to know who we are talking to aka which element in our HTML structure is actually adressed before doing anything. We can do this by »getting« and element. This is a concept widely used in programming: getting and setting. This simply means that we can get information from an element but also change the element. This is referred to as »setting«.
OK, considering a real-world example and let us start with a simple exaple to change the font-variation-settings of our fonts. So, say we have a text, that we want to change the variable font within on clicking on it. The HTML is pretty straight forward and looks as follows:
<div id="sample-text">
Here comes the text
</div>
 We could now directly manipulate the font by using JavaScript:
document.getElementById("sample-text").style.setProperty("font-variation-settings","'VROT' 30","");
 With the first section we are getting the element with the ID sample-text then we are setting the style to the property named in the brackets. This for now is not really interesting as we could more easily achieve the same by using CSS. However, if we mix in some interactivity the JavaScript part becomes a bit more interesting:
document.getElementById("sample-text").addEventListener("click", function () {
document.getElementById("sample-text").style.setProperty("font-variation-settings","'VROT' 30","");
});
 We are once more getting the element sample-text with the document.getElementById("sample-text") part. With the following part we are adding a so called event listeners. These are like little helpers, that can help us watch out for events of all sorts: clicks, double clicks, mouse releases, key presses etc. and fire events once the event happens. In our case change the font to the defined settings.
But, I hear you say, this is a one way. Once fired the event will stay as is until the page gets reloaded. There must be a nicer way to achieve the same. (Note in programming there often ist not one single way to achieve result xy but multiple). So let’s try something else:
document.getElementById("sample-text").addEventListener("click", function () {
document.getElementById("sample-text").classList.toggle("vfont");
});
 In this case we use a helper class named vfont which could look as follows:
.vfont {
font-variation-settings: 'VROT' 30;
transition: 0.3s;
}
 The idea here is simple: if the class is active, we remove it (and the font returns to normal, whatever this is). If it is not, the class gets applied, so transitioned to the settings defined in the vfont class. Last but not least, we could add a transtion to the sample-text ID and our font wouuld nicely animate on click. Yikes — we wrote our first lines of JavaScript!
